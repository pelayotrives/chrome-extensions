# Changelog

All notable changes to Extensio Patronus are documented in this file.

This project now tracks releases with the newest entry first for easier maintenance.

---

## [0.1.1]

### Added

- Faster snapshot creation from the `Extensions` tab with a dedicated `Snapshot` shortcut button beside sorting.
- Destructive restore-default flow that now also clears saved aliases, alongside groups, supergroups, snapshots, and assignments.
- Duplicate-name protection with danger toasts for creating or renaming groups, supergroups, and snapshots.

### Changed

- Snapshot primary action label is now `Snapshot` both in the `Snapshots` tab and in the `Extensions` shortcut for a more consistent CTA.
- Group and supergroup cards now prioritize existing chips and toggles, while creation actions sit lower in the card for a more task-first UX.
- `How it works` and `Keyboard shortcuts` now use the same title styling as `Settings`.
- `Power colors` now lives inside `Keyboard shortcuts` as a subsection.
- `Avada Kedavra` copy now frames the destructive reset as restoring the default setup.
- Snapshot auto-naming now skips existing names so autogenerated labels do not collide with renamed snapshots.
- Manifest version bumped to the latest patch release.

## [0.1.0]

### Changed

- Snapshot empty state now matches the shared dashed empty-state layout used by groups and supergroups.
- Version bumped to the first minor release and now appears in the popup footer through the existing manifest-driven label.

---

## [0.0.10]

### Changed

- Supergroup empty state now uses a centered dashed placeholder style with muted section-subtitle typography.

### Fixed

- Restored the new snapshot shine animation after the import-flow styling regressions.
- Fixed the import overlay spinner layout so the progress ring stays centered inside a proper modal overlay instead of spilling across tabs.
- Hardened `renderSupergroups()` exposure so Chrome no longer throws runtime `ReferenceError` failures during popup init, imports, drag and drop, or assignment flows.

---

## [0.0.9]

### Added

- Freeform group and supergroup usage with only a 9-snapshot cap.
- Red top toast for snapshot-limit attempts.
- Success and danger top toasts for create, delete, import, export, enable, disable, assign, and error flows.
- Right-click supergroup edit mode using the existing form.
- Toast when entering supergroup edit mode.
- Snapshot shine animation on the newly created snapshot row.
- Micro empty states for groups and supergroups.
- Commercial README, dedicated privacy policy, restrictive license file, and store listing copy pack.

### Changed

- Extension is now fully free and no longer shows licensing or plan UI.
- Footer authorship and rights messaging refined.
- `How it works` and `Keyboard shortcuts` headings now match the Settings heading style.
- Manifest description updated for store readiness.
- Snapshot shine animation slowed down and eased more naturally.
- Changelog versioning reset to patch-only release history.

### Fixed

- License tab rendering issues removed by removing the entire licensing flow.
- Snapshot save flow now shows the limit toast instead of a persistent inline limit hint.
- Group tooltip now mentions both filtering and renaming.
- Supergroup tooltip now documents filtering, renaming, and right-click editing together.
- Tabs now expose better ARIA state with `role="tab"` / `role="tabpanel"` semantics.
- Toasts now expose `role`, `aria-live`, and `aria-atomic` attributes for assistive technologies.
- Inline status text for group-related messages is now announced politely.
